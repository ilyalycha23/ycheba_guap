# Лабораторная работа №1
## Концептуальная модель IoT-системы в виде UML-диаграммы

**Вариант:** Умное игровое поле для игры в баскетбол

---

### Цель работы
Разработать концептуальную модель IoT-системы в соответствии с выбранным вариантом.

---

### Задание
1. В виде текста (или рисования) описать концептуальную модель IoT-системы, состоящую из трёх слоёв (рис. 1).
2. Разработать UML-диаграмму классов Thing и Sensor.

---

### Теоретическая часть

#### Трёхслойная архитектура IoT-системы

1. **Физический слой (Physical Layer)** — нижний уровень, включает физические устройства: датчики попаданий мяча в кольцо, датчики линий площадки, освещение, табло счёта.

2. **Слой цифрового двойника (Digital Twin Layer)** — слой, где виртуальная модель отражает состояние физической площадки: счёт игры, статистика бросков, состояние освещения.

3. **Слой пользовательского интерфейса (User Interface Layer)** — слой взаимодействия: веб-интерфейс, мобильное приложение для отображения счёта и статистики.

---

### Ход работы

#### 1. Класс Thing (абстрактный)
Базовый абстрактный класс для всех IoT-объектов умного баскетбольного поля.

```
+------------------+
|      Thing       |
+------------------+
| - name: string   |
+------------------+
| + connect()*     |
+------------------+
```

#### 2. Класс Sensor (наследует Thing)
Датчик попаданий мяча в корзину — фиксирует успешные броски.

```
+---------------------------+
|         Sensor            |
+---------------------------+
| - unit: string            |
| - value: int              |
| - name: string (from Thing)|
+---------------------------+
| + connect()               |
| + emulate()               |
+---------------------------+
```

#### 3. Диаграмма наследования
Sensor наследует от Thing (абстрактный класс).

#### 4. Диаграмма взаимодействия
Sensor взаимодействует с MainControlUnit (центральный контроллер, обрабатывающий данные датчиков и управляющий табло счёта).

---

### Структура системы "Умное игровое поле"
- **ShotSensor** — датчик попаданий (количество очков за попадание)
- **MainControlUnit** — центральный контроллер (обработка счёта)
- **Scoreboard** — актуатор (отображение счёта на табло)

2. Определены основные классы: Thing (абстрактный базовый), Sensor (датчик попаданий).
