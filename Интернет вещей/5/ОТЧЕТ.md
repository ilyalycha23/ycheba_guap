# Лабораторная работа №5
## Обработка данных датчика с валидацией

**Вариант:** Умное игровое поле для игры в баскетбол

---

### Цель работы
Добавить валидацию входящих данных датчика с использованием блока try-except.

---

### Задание
Реализовать валидацию данных: принимать только числовые значения (1, 2 или 3 — допустимые очки в баскетболе). При некорректном вводе обрабатывать исключение и не обновлять значение.

---

### Ход работы

1. **Блок try-except** в методе `connect`:
   - `try`: преобразование `value` в `float()` и `int()`, проверка на допустимость (1, 2 или 3).
   - `except ValueError`: вывод сообщения в консоль, возврат JSON с `power: "error"`.

2. **Валидация:**
   - Допустимые значения: 1, 2, 3 (очки за обычный бросок, за бросок из зоны, трёхочковый).
   - Недопустимые: нечисловые строки, числа вне диапазона.

3. **Поведение:** при успешной валидации значение обновляется и возвращается `power: "on"`; при ошибке — `power: "error"`.

---

### Выводы
1. Реализована валидация входящих данных с использованием try-except.
2. Система корректно обрабатывает некорректный ввод.
3. Допустимые значения ограничены правилами баскетбола (1, 2, 3 очка).
