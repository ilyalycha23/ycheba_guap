# Лабораторная работа №7
## Логгер с сохранением в MongoDB

**Вариант:** Умное игровое поле для игры в баскетбол

---

### Цель работы
Реализовать класс Logger для сохранения данных датчика в MongoDB.

---

### Задание
Добавить логгер, который сохраняет очки попаданий в MongoDB при каждом успешном обновлении значения датчика.

---

### Ход работы

1. **Установка MongoDB и pymongo:**
   - MongoDB: https://www.mongodb.com/docs/manual/installation/
   - `pip install pymongo`

2. **Класс Logger:**
   - Подключение: `pymongo.MongoClient('mongodb://localhost:27017/')`
   - Метод `insert_score(new_data)` — сохраняет очки в коллекцию `Scores` при изменении значения
   - Документ: `timeStamp`, `Points`

3. **Условие записи:** запись выполняется только если значение изменилось (`new_data != self.current_score`).

4. **Интеграция:** после успешной валидации и обновления датчика вызывается `logger.insert_score(shot_sensor.value)`.

---

### Выводы
1. Реализован логгер с сохранением в MongoDB.
2. Данные сохраняются только при изменении значения.
3. Система готова к отображению истории (лаба 8).
