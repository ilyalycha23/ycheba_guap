Лабораторная работа №2 (п. 2.1–2.5 методички). Вариант 12. СУБД: PostgreSQL.

Порядок выполнения (в пустой БД или после пересоздания таблиц):
1) 01_schema.sql — если таблицы ещё не созданы (как в ЛР1).
2) 02_insert_test_data.sql — тестовые данные (п. 2.1, п. 2.5).
3) 03_select_datasets.sql — просмотр содержимого таблиц (для отчёта: «наборы данных»).
4) 04_examples_invalid.sql — примеры вставок, нарушающих check / unique / FK (должны завершиться ошибкой).
5) 05_update_delete.sql — корректировки update/delete (п. 2.2, 2.3).
6) 06_cascade_and_alter.sql — пример on update cascade; демонстрация on delete cascade на временных таблицах (п. 2.5); alter table (п. 2.4).

Файл 04_examples_invalid.sql: выполняйте операторы по одному (каждый должен завершиться ошибкой).

В отчёт по п. 2.5 приложите: схему (если меняли), результаты select, фрагменты скриптов с комментариями, что является «некорректной» вставкой и какая ошибка ожидается.
