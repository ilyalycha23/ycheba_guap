# Пояснения к лабораторным работам (курс «Методы и средства проектирования»)

Здесь собраны **подробные объяснения** к скриптам из папок `lab1`–`lab8`: зачем нужна каждая лабораторная, в каком порядку читать файлы и **что означает код** — по строкам, либо по логическим фрагментам (для повторяющихся конструкций).

## Как пользоваться

1. Сначала прочитайте **краткий обзор лаборатории** в соответствующем файле `ЛР##_….md`.
2. Откройте **исходный `.sql`** в Cursor / pgAdmin рядом с пояснением.
3. В пояснениях номера строк относятся к версии файлов в репозитории на момент создания гайда; если вы меняли скрипт, ориентируйтесь по содержимому, а не только по номеру.

## Логическая цепочка курса

- **ЛР1** задаёт **модель данных** (таблицы и связи «предприятие»).
- **ЛР2** наполняет базу **фактами** и показывает простейший **DML**.
- **ЛР3** — **SELECT** без подзапросов и агрегатов (фильтры, соединения в WHERE).
- **ЛР4** — **агрегаты** и **операции над множествами** результата (UNION и т.д.).
- **ЛР5** — **коррелированные подзапросы** с **EXISTS**, DML с подзапросами, **NULL**.
- **ЛР6** — серверная логика: **процедуры и функции** в PostgreSQL.
- **ЛР7** — **триггеры** (автоматические действия на INSERT/UPDATE/DELETE).
- **ЛР8** — **представления**, **индексы**, **планы** запросов, опционально **права**.

Файлы `lab2/main_schema.sql`, `lab3/main_schema.sql`, `lab4/main_schema.sql` в гайде не разбираются отдельно: как правило, это **копия** схемы для удобства сдачи; опора — на `lab1/01_schema.sql`.
