## `01_queries_zhi.sql` — построчная логика

### ж) «Нет совместителей в подразделении» (строки 5–16)

```sql
WHERE NOT EXISTS ( SELECT 1 FROM прием пр WHERE ... AND пр.совместительство = B'1' )
```

- Внешний цикл: каждая строка `подразделение пд`.
- **NOT EXISTS (подзапрос)**: «не существует ни одной** строки `прием` для этого `пд.ид` с `совместительство = 1 (бит)».
- `SELECT 1` в `EXISTS` — стандарт: важно **факт** существования, а не конкретное значение; часто пишут `SELECT 1` или `SELECT NULL::int`.

**Почему не COUNT = 0?** Можно и через `LEFT JOIN` + `HAVING`, но `NOT EXISTS` часто **прозрачнее** и с хорошим планом.

### з) «Все должности 1–4 в штате» (строки 21–36)

Двойное **NOT EXISTS** (квантор «для всех»):

1. «Не существует такой **должности из 1,2,3,4** …»
2. «…для которой **не** существует строки **штатного_расписания** в этом подразделении с этой должностью».

То есть: для **каждой** требуемой должности должна найтись **хотя бы одна** штатная строка `(подразделение, должность)`.

**Важно:** на «голых» данных ЛР2 в подр. 3 **не хватает** строк штата по всем 1–4 — поэтому **перед** запросом выполняют `04_extra_data_lr5.sql` (см. ниже).

### и) «Все дети одного пола» (строки 40–58)

- `EXISTS (дети)` — **есть** хотя бы один ребёнок.
- `NOT EXISTS (д1 CROSS JOIN д2 WHERE ... d1.пол <> d2.пол)` — **нет** пары **разных** детей с **разным** полом **у этого** сотрудника.
- `d1.ид < d2.ид` — **устраняет** двойной счёт пар (A,B) и (B,A) и отсекает `d1 = d2`.
- **CROSS JOIN** внутри `NOT EXISTS` — допустим: пары **все** комбинации, фильтр оставляет только «разнополых» (если кто-то не любит CROSS — эквивалент можно писать через сравнение детей с `д1.ид <> д2.ид`).

## `03_exists_vs_set_nulls.sql`

- **Первый `SELECT`:** пересечение — сотрудник в приёме в **2** и **существует** приём в **4** — формула как `INTERSECT` из ЛР4, но в терминах **коррелированного** `EXISTS`.
- **Второй:** `NOT EXISTS (увольнение)` = «как `EXCEPT`» из `прием` минус `увольнение`.
- **NULL-демо:** `WHERE x IS NULL` находит `NULL` в `VALUES (NULL::int), (1)`; `WHERE x = NULL` в SQL **не** даёт TRUE для NULL (трехзначная логика) — **строка не отбирается**; `NOT (NULL = NULL)` тоже не TRUE — **никогда не используйте `= NULL` для проверки null**.

## Сводка «зачем EXISTS»

- **EXISTS** останавливается, как только найдена **первая** подходящая строка (часто эффективно).
- **INTERSECT** требует **сортировки/хеша** обоих множеств; при сложных условиях **EXISTS** читабельнее.
- **NULL** требуют `IS [NOT] NULL`.
