# Лабораторная 6 — хранимые процедуры и функции (PostgreSQL)

## Идея

В **ЛР1–5** логика была в **клиентских** SQL-скриптах. В **ЛР6** часть логики переносится **на сервер**:

- **`CREATE PROCEDURE`** + **`CALL`** — процедуры (в PostgreSQL с версии 11).
- **`CREATE FUNCTION`** — скалярные и табличные функции.
- **`TEMP TABLE`** — временные таблицы **в сессии**.
- **`DO $$ ... $$`** — анонимный блок plpgsql (аналог «пакета» из методички по Oracle — с оговорками).

Префикс имён **`lr6_`** — чтобы не пересекаться с учебными таблицами.

## `04_fn_aggregate.sql`

| Строки | Пояснение |
|--------|-----------|
| 3 | `CREATE FUNCTION` … `RETURNS bigint` — **скалярная** функция. |
| 4 | `LANGUAGE sql` — тело = **один** SQL-запрос (без BEGIN/END plpgsql), компактно. |

## `08_fn_table.sql`

- `RETURNS TABLE(ид int, наименование varchar(200))` — функция, которую пишут как **`SELECT * FROM lr6_direct_subdepts(1)`** — табличный результат.
- `LANGUAGE sql` + один `SELECT` — в PostgreSQL **SET-returning** в SQL-функции разрешён.
- Условие `WHERE` реализует: либо **корневые** (оба NULL), либо **дети** конкретного родителя. **`IS NULL`** важен для сравнения с NULL.

## `09_drop_lr6_objects.sql`

- Для `DROP PROCEDURE` в PostgreSQL **нужны сигнатуры** `(типы)` — перегрузка по типам.
- `DROP FUNCTION` с `(integer)` — одна аргументная функция.
- **Порядок:** сначала процедуры/функции, не забыв зависимости (тут зависимостей между lr6-объектами в другом направлении — ок).

## `10_demo_transaction.sql`

- `BEGIN` — начало транзакции.
- `CALL` вставки — **видна** внутри той же транзакции.
- `INSERT` тестового подразделения + `CALL` удаления листа.
- `ROLLBACK` — **всё** откатится, включая вставки в `должность`, `прием`, `подразделение` (если не закоммитили) — **удобно** для сдачи, не портя основные тестовые данные.

## Краткий глоссарий

- **Процедура** — хранимый `CALL`, может не возвращать таблицу клиенту (в отличие от «табличной» функции).
- **Функция SQL** — часто `STABLE`/`IMMUTABLE` для кэширования планов; тело **без** plpgsql, если логика одной формулой.
- **TEMP** — сессия; не **глобальная** статистика для всех.
